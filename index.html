Claude Code 설치 가이드

본 교안은 AI 에이전트에게 업무를 위임하는 실습 교육에 참여하시는 분들을 위한 사전 자가 설치 자료입니다. 터미널을 처음 열어보시는 분도 순서대로 체크박스만 따라가시면 교육 당일 실습 환경이 완성됩니다.

예상 소요 60–90분 대상 완전 초보 ~ 중급 지원 OS macOS · Windows 발행일 2026. 04

이 박스는 수강생이 아닌 교육 주관자를 위한 것입니다

본 교안을 수강생에게 배포하시기 전에, 두 가지 사항을 먼저 결정해 주세요. 결정 내용에 따라 수강생에게 안내할 계정 가입 방법이 달라집니다. Claude Code의 실습은 각 수강생의 로컬 PC에서 이루어지지만, AI 호출은 Anthropic 서버로 이뤄지므로 구독 계정이 반드시 있어야 합니다.

Part 0 · Before You Start

00시작하기 전에

본격적인 설치에 앞서 꼭 준비하셔야 할 것과, 이 교안을 읽는 방법을 먼저 안내합니다. 용어가 낯설어도 괜찮습니다. Part 4의 용어집에 모두 정리되어 있으니, 막히실 때마다 넘어가서 확인하시면 됩니다.

준비물

  • 안정된 인터넷 연결 (설치 파일 다운로드에 약 1~3GB가 필요합니다)
  • 본인 관리자 권한이 있는 노트북 또는 데스크탑 (회사 지급 PC의 경우 IT 담당자에게 설치 권한 요청이 필요할 수 있습니다)
  • 여유 저장공간 최소 5GB
  • 본인 명의의 이메일 계정 (회사 이메일 권장)
  • 결제용 신용카드 또는 해외결제 가능 체크카드 (Claude 유료 구독 결제용)

이 교안을 따라가시는 법

읽기 전 확인. 상단의 macOS / Windows 탭을 먼저 선택해 주세요. 본인 OS에 해당하는 설명만 보이도록 자동으로 필터링됩니다. 체크박스를 클릭하시면 진행 상황이 브라우저에 자동 저장되어, 창을 닫았다 다시 열어도 이어서 하실 수 있습니다.
터미널이라는 단어가 낯설다면. 앞으로 자주 나올 "터미널"은 컴퓨터에 명령어를 직접 입력하는 검은(또는 흰) 창입니다. macOS에서는 ⌘ Cmd + Space를 눌러 "터미널"을 검색해 열고, Windows에서는 시작 메뉴에서 "PowerShell"을 검색해 관리자 권한으로 실행합니다. 이 교안의 모든 명령어는 그 창에 붙여넣고 Enter를 누르시면 됩니다.
Part 1 · Prerequisites

1.1 Node.js 설치

아래 공식 사이트 링크에 접속하면 자동으로 macOS용 LTS 버전(현재 22.x 또는 24.x 'Krypton')이 표시됩니다. "macOS Installer(.pkg)"를 눌러 다운로드합니다.

👉 https://nodejs.org/en/download

다운로드가 끝나면 .pkg 파일을 더블클릭하여 실행하고, 안내에 따라 "계속" → "동의" → "설치"를 눌러 완료합니다. 관리자 비밀번호를 요구하면 Mac 로그인 비밀번호를 입력하시면 됩니다.

터미널을 새로 열고(⌘ + Space → "터미널" 검색) 아래 명령어를 붙여넣은 뒤 Enter를 누릅니다.

$ node --version $ npm --version

각각 v22.x.x 이상, 10.x.x 이상과 비슷한 버전 번호가 나오면 성공입니다.

아래 공식 사이트에서 Windows Installer(.msi)를 다운로드합니다. 보통 자동으로 LTS 버전(현재 22.x 또는 24.x 'Krypton')이 추천됩니다.

👉 https://nodejs.org/en/download

다운로드한 .msi 파일을 더블클릭하여 실행합니다. 설치 마법사에서 모든 옵션은 기본값 그대로 두고 "Next"를 계속 누르면 됩니다.

⚠️ 중요. 설치 도중 "Automatically install the necessary tools..." (추가 도구 자동 설치) 체크박스가 나오면 반드시 체크한 뒤 진행해 주세요. 이 옵션이 C++ 빌드 도구를 함께 설치하여 이후 MCP 설치 시 오류를 예방합니다.

시작 메뉴에서 "PowerShell"을 검색해 실행합니다. 아래 명령어를 붙여넣고 Enter를 누릅니다.

PS> node --version PS> npm --version

각각 v22.x.x 이상, 10.x.x 이상 버전이 출력되면 성공입니다.

💡 TIP. PowerShell 창에서 붙여넣기는 Ctrl + V가 아니라 마우스 오른쪽 버튼 클릭 또는 Shift + Insert입니다.

1.2 Git 설치

macOS에는 Apple이 제공하는 개발자 도구 패키지에 Git이 포함되어 있습니다. 터미널을 열고 아래 명령어 한 줄을 실행하시면 됩니다.

$ xcode-select --install

작은 창이 뜨면서 "도구 설치" 여부를 묻습니다. "설치"를 눌러 진행해 주세요. 설치는 자동으로 진행되며 5~15분 정도 걸립니다.

이미 설치되어 있다는 메시지가 나온다면? xcode-select: error: command line tools are already installed 같은 메시지는 "이미 잘 설치되어 있다"는 뜻이니 그대로 진행하시면 됩니다.
$ git --version

git version 2.xx.x처럼 버전 번호가 표시되면 성공입니다.

Windows에서는 Git for Windows라는 별도 설치 프로그램이 필요합니다. Claude Code가 내부적으로 이 Git에 포함된 Git Bash를 사용하기 때문에 반드시 설치하셔야 합니다.

👉 https://git-scm.com/downloads/win

"64-bit Git for Windows Setup"을 클릭해 다운로드합니다.

다운로드한 .exe 파일을 더블클릭하여 실행합니다. 설치 옵션이 매우 많지만 모두 기본값 그대로 "Next"를 누르시면 됩니다. 마지막에 "Install"을 눌러 설치를 완료합니다.

⚠️ 설치 후 반드시. PowerShell 창이 열려 있었다면 모두 닫고 새로 여세요. 새로 열지 않으면 git 명령어가 인식되지 않습니다.

PowerShell을 새로 열고 아래를 입력합니다.

PS> git --version

git version 2.xx.x.windows.x와 비슷한 결과가 나오면 성공입니다.

1.3 Python 설치

공식 사이트에서 macOS 64-bit universal2 installer를 받으시면 됩니다. 2026년 4월 기준 최신 안정 버전은 Python 3.14.x입니다.

👉 https://www.python.org/downloads/macos/

내려받은 .pkg 파일을 더블클릭해 설치합니다. 옵션은 모두 기본값으로 두고 진행하시면 됩니다.

터미널에서 아래 명령어로 버전을 확인합니다. macOS에서는 "python"이 아닌 "python3"을 사용해야 합니다.

$ python3 --version $ pip3 --version

Python 3.14.x(또는 3.13.x) 및 pip 25.x가 나오면 성공입니다.

공식 사이트에서 Windows installer (64-bit)을 받습니다. 2026년 4월 기준 최신 안정 버전은 Python 3.14.x입니다.

👉 https://www.python.org/downloads/windows/

⚠️ 절대 놓치면 안 되는 한 가지. 설치 프로그램을 실행하면 첫 화면 아래쪽에 "Add python.exe to PATH"(또는 "Add Python to environment variables") 체크박스가 있습니다. 반드시 이 체크박스를 먼저 클릭한 뒤 "Install Now"를 눌러주세요. 이것을 빠뜨리면 터미널에서 python 명령어를 인식하지 못합니다.

설치가 끝나면 "Disable path length limit" 버튼이 보이면 한 번 눌러 활성화한 뒤 창을 닫습니다.

PowerShell을 새로 열고 아래를 입력합니다.

PS> python --version PS> pip --version

Python 3.14.x, pip 25.x 같은 결과가 나오면 성공입니다.

2.2 Claude Code 설치

2025년 10월부터 Anthropic은 Node.js가 필요 없는 Native Installer를 기본 설치 방식으로 권장하고 있습니다. 우리는 이미 Node.js를 설치했지만, 자동 업데이트가 지원되고 설정이 더 단순한 Native 방식으로 설치하겠습니다.

그럼 Node.js는 왜 미리 설치했을까요? Claude Code 자체를 실행하는 데는 Node.js가 필요하지 않습니다. 하지만 본 교육 이후 여러분이 Claude Code를 실제 업무에 활용하기 시작하면, 수많은 보조 도구·MCP 서버·플러그인이 Node.js 프레임워크 위에서 동작합니다. 바로 우리가 Part 3에서 설치할 Playwright MCP와 Context7 MCP도 모두 Node.js의 npx 명령으로 실행되고, 앞으로 마주할 거의 모든 확장 도구가 같은 구조입니다. 따라서 Node.js는 Claude Code 본체를 위한 것이 아니라, "앞으로 Claude Code와 함께 쓰게 될 개발 생태계 전체"의 기반으로 미리 깔아 두는 것입니다. 처음부터 설치해 두어야 실습이 끊김 없이 진행됩니다.

터미널을 열고 아래 명령어를 붙여넣은 뒤 Enter를 누릅니다. 설치에 1~2분 정도 걸립니다.

$ curl -fsSL https://claude.ai/install.sh | bash

설치가 끝나면 터미널 창을 완전히 닫고 다시 열어 주세요. 그래야 claude 명령어가 인식됩니다.

$ claude --version

2.x.x (Claude Code)와 같은 버전이 표시되면 성공입니다.

⚠️ 반드시 PowerShell에서. CMD(명령 프롬프트)가 아닌 PowerShell에서 실행해야 합니다. 화면에 PS C:\Users\...> 처럼 맨 앞에 "PS"가 붙어 있는지 확인해 주세요.

처음 한 번만, 설치 스크립트 실행이 허용되도록 아래 명령어를 실행합니다(한 번만).

PS> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

"변경하시겠습니까" 질문에 Y를 입력하고 Enter. 이어서 설치 명령을 실행합니다.

PS> irm https://claude.ai/install.ps1 | iex

설치가 끝나면 PowerShell 창을 완전히 닫고 새로 열어 주세요.

PS> claude --version

2.x.x (Claude Code)와 같은 버전 번호가 나오면 설치 성공입니다.

ℹ️ 더 자세한 진단이 필요할 때. claude doctor 명령어를 실행하면 환경이 제대로 구성됐는지 종합적으로 점검해 줍니다.

2.3 최초 실행 및 로그인

우선 실습에 사용할 빈 폴더를 홈 디렉토리에 만들고 그 안으로 이동합니다.

$ mkdir ~/claude-practice $ cd ~/claude-practice $ claude
PS> mkdir $HOME\claude-practice PS> cd $HOME\claude-practice PS> claude

Part 3 · MCP Servers

03기본 MCP 설정 — Playwright · Context7

MCP(Model Context Protocol)는 Claude가 외부 도구·서비스와 연결되어 일을 "직접 수행"할 수 있도록 해주는 표준 통신 규격입니다. 여기서는 본 교육에서 사용할 두 가지 기본 MCP를 설정합니다. 자세한 개념 설명은 Part 4 용어집에서 다룹니다.

3.1 Playwright MCP — 웹 브라우저 자동 조작

Playwright는 원래 Microsoft가 만든 브라우저 자동화 도구입니다. 이 MCP를 연결하면 Claude가 실제 크롬 브라우저를 열어 웹 사이트 방문, 클릭, 양식 입력, 스크린샷 캡처 등을 사람처럼 수행할 수 있습니다. 예를 들어 "경쟁사 홈페이지에 접속해서 가격표를 표로 정리해줘" 같은 업무 위임이 가능해집니다.

터미널 또는 PowerShell을 열고 아래 명령어를 입력합니다. Claude Code 인터랙티브 창 밖에서 실행하셔야 합니다.

$ claude mcp add playwright --scope user -- npx -y @playwright/mcp@latest
옵션의 의미. --scope user는 이 MCP를 모든 폴더에서 사용할 수 있게 전역 설정한다는 뜻입니다. -y는 자동으로 "yes"를 눌러 설치를 진행합니다.
$ claude mcp list

출력 결과에 playwright 줄이 포함되고 상태가 connected 또는 ok로 표시되면 성공입니다. 최초 사용 시 브라우저 엔진이 자동 다운로드되므로 1~2분 소요될 수 있습니다.

3.2 Context7 MCP — 최신 기술 문서 자동 참조

Context7은 각종 프로그래밍 라이브러리의 최신 공식 문서를 Claude에게 자동으로 공급해 주는 MCP입니다. AI는 학습 시점 이후의 최신 정보를 모르지만, Context7을 연결하면 Claude가 "use context7"이라는 말 한마디만 붙여도 그때그때 공식 문서를 찾아 최신 코드 예제로 답하게 됩니다. 교육에서 다룰 여러 도구·프레임워크 실습 시 매우 유용합니다.

아래 사이트에 접속해 GitHub 또는 Google 계정으로 간단히 가입하시면 무료 API 키를 받으실 수 있습니다. 키가 없어도 동작하지만 있을 때 속도 제한이 훨씬 여유롭습니다.

👉 https://context7.com/dashboard

발급받은 키는 아래 명령어에서 YOUR_API_KEY 자리에 붙여넣게 됩니다. 키는 외부에 공개하지 마세요.

아래 명령어의 YOUR_API_KEY 부분을 본인 키로 바꿔서 실행합니다.

$ claude mcp add context7 --scope user --transport http \ --header "CONTEXT7_API_KEY: YOUR_API_KEY" \ https://mcp.context7.com/mcp
💡 API 키 없이 설치하려면? 아래 명령어는 키 없이 바로 붙여 사용할 수 있습니다(요청량 제한이 있음).
$ claude mcp add context7 --scope user --transport http https://mcp.context7.com/mcp
$ claude mcp list

목록에 playwright와 context7 두 개가 모두 있고, 각각 connected 상태라면 Part 3 완료입니다.

Part 4 · Glossary

04꼭 알고 오셔야 할 용어집

교육 현장에서 시간을 아끼기 위해, 자주 등장할 용어들을 미리 정리해 드립니다. 완벽히 이해하지 않으셔도 괜찮습니다. 각 용어의 "한 줄 정의"만이라도 머릿속에 넣고 오시면 실습 내용을 훨씬 수월하게 따라오실 수 있습니다.

에이전트 Agent

"스스로 판단해서 일하는 AI"를 말합니다. 사용자가 목표만 알려 주면, 에이전트는 어떤 순서로, 어떤 도구를 써서, 어떻게 처리할지를 스스로 계획하고 실행합니다. 결과를 확인해 문제가 있으면 다시 시도하기도 합니다.

예) "이번 주 경쟁사 블로그 새 글을 모두 찾아 요약해줘"라고 하면, 에이전트는 검색 → 접속 → 읽기 → 요약 → 정리를 순서대로 알아서 수행합니다.

LLM 채팅 vs LLM 에이전트 Chat vs Agent

쉽게 비유하자면 이렇습니다.

  • LLM 채팅 서비스 — "똑똑한 조언자." 질문하면 답해주지만, 실제로 일은 하지 않습니다. 결과물을 얻으려면 사용자가 직접 그 답을 복사해 어딘가에 붙여넣어야 합니다. (예: ChatGPT 기본 채팅창, Claude 웹 채팅)
  • LLM 에이전트 — "실제 일해주는 비서." 컴퓨터의 파일을 직접 열고, 웹에 접속하고, 메일을 작성해 보내기까지 합니다. 사용자는 목표만 제시합니다. (예: Claude Code, Claude with MCP)

본 교육의 핵심은 "조언받기"에서 "위임하기"로의 전환입니다.

MCP Model Context Protocol

Claude 같은 AI 에이전트가 외부 도구·서비스와 대화하는 공통 규격입니다. 사람에게 비유하자면 "만능 콘센트 규격"과 같습니다. MCP라는 표준이 있기에, 전혀 다른 회사가 만든 브라우저 자동화(Playwright), 최신 기술 문서(Context7), 메일(Gmail), 캘린더(Google Calendar) 같은 도구를 똑같은 방식으로 Claude에 꽂아 쓸 수 있습니다.

MCP 하나를 "서버(Server)"라고 부르며, 여러 MCP를 동시에 연결해 쓸 수 있습니다. 우리는 Part 3에서 Playwright, Context7 두 개를 연결했습니다.

ACP Agent Communication Protocol

서로 다른 AI 에이전트끼리 대화할 수 있게 해주는 표준입니다. MCP와 한 글자 차이라 혼동되기 쉬운데, 역할은 다음처럼 명확히 나뉩니다.

  • MCP — "AI ↔ 도구" 통신. (Claude가 브라우저를 조작하거나 메일을 보내는 경로)
  • ACP — "AI ↔ 다른 AI" 통신. (한 조직의 Claude 에이전트와, 다른 회사의 LangChain이나 CrewAI로 만든 에이전트가 서로 협력하는 경로)

IBM의 BeeAI 프로젝트에서 처음 제안되었고, 현재는 Linux Foundation 산하에서 A2A(Agent-to-Agent Protocol)와 통합되는 과정에 있습니다. 아직 초기 단계의 표준이라 실무에서는 MCP만큼 널리 쓰이지는 않지만, 앞으로 여러 회사·플랫폼의 AI가 서로 협업하는 "에이전트 웹(Agentic Web)"이 본격화되면 핵심 인프라가 될 것으로 기대되는 규격입니다.

교육 실습에서는 MCP만 다루므로 "아, 서로 다른 AI끼리 통신하는 규격이 따로 있구나" 정도만 머리에 두시면 충분합니다.

Skills Skills

Claude가 특정 작업을 할 때 자동으로 불러오는 "매뉴얼 폴더"입니다. 각 Skill은 SKILL.md라는 설명서 파일과 필요한 보조 파일들로 구성됩니다.

예를 들어 "이 데이터로 엑셀 보고서 만들어줘"라고 하면, Claude는 알아서 xlsx Skill을 꺼내 보고 그 안에 적힌 순서대로 작업합니다. Skill의 설명(description)에 적힌 트리거 문구가 사용자 요청과 매칭되면 자동으로 발동됩니다.

회사 고유 업무 방식(예: "우리 회사 견적서 양식")을 Skill로 만들어 두면, 모든 직원이 같은 수준의 결과물을 받게 됩니다.

메인 에이전트 Main Agent

사용자와 직접 대화하는 "중심 AI". 지시를 받아 전체 작업을 계획하고, 필요에 따라 서브 에이전트에게 일을 나눠 맡기고, 마지막에 결과를 종합해 사용자에게 보고하는 "현장 책임자"입니다.

본 교육의 실습 환경에서는 여러분이 Claude Code를 실행했을 때 처음 대화하는 그 AI가 메인 에이전트입니다.

서브 에이전트 Sub-agent

메인 에이전트가 특정 작업을 위해 별도로 호출하는 "전문가 AI". 각 서브 에이전트는 자신만의 독립된 작업 공간(컨텍스트)을 가지고, 맡은 일을 처리한 뒤 결과 요약만 메인에게 돌려줍니다.

왜 나누는가?

  • 집중력 보존. 메인 에이전트가 30개의 파일을 모두 읽으면 혼란스러워지지만, "코드 리뷰어" 서브 에이전트에게 맡기면 메인은 결론만 받습니다.
  • 전문성 분리. "세무 전문가", "디자인 전문가"처럼 역할별로 튜닝할 수 있습니다.
  • 권한 제한. 검토만 하는 서브 에이전트는 파일 쓰기 권한을 빼서 안전하게 운용할 수 있습니다.
에이전트 팀 Agent Team

여러 AI 에이전트가 서로 대화하며 협업하는 방식. 서브 에이전트는 단방향(지시→결과)인 반면, 에이전트 팀은 작업 중간에 서로 의견을 나누고 조정합니다.

예를 들어 공공입찰 전략을 짤 때, "공무팀 담당", "재무 담당", "설계 담당" 에이전트들이 한 테이블에서 회의하면서 최종 제안을 함께 만들어 냅니다. 실제 회사 조직도를 그대로 옮겨 놓은 것과 같습니다.

강력하지만 토큰 소모가 크므로, 교차 협의가 꼭 필요한 복잡한 업무에만 사용하는 것이 원칙입니다.

오케스트레이션 Orchestration

여러 에이전트·도구·단계를 지휘자처럼 조율하여 하나의 큰 업무를 완성시키는 것을 말합니다. 오케스트라의 지휘자가 현악기·관악기·타악기에게 "누가, 언제, 어떻게" 연주하라고 지시하는 장면을 떠올리시면 됩니다.

예를 들어 "이번 분기 공공입찰 대응 전략을 만들어줘"라는 지시를 받으면, 메인 에이전트는 다음과 같이 일을 쪼개어 배치합니다.

  • 리서치 서브 에이전트 → 나라장터에서 관련 공고·경쟁사 정보 수집
  • 재무 서브 에이전트 → 원가·이익률 분석
  • 문서 작성 서브 에이전트 → 제안서 초안 작성
  • 검토 서브 에이전트 → 오탈자·논리 허점 점검

이 네 명에게 어떤 순서로, 어떤 데이터를 넘기며, 언제 병렬로 돌릴지를 설계하고 실행하는 것이 오케스트레이션입니다.

전통적으로는 개발자가 코드로 이 흐름을 미리 짜 두어야 했지만, 최신 Claude Code는 사용자가 목표만 제시하면 Claude가 스스로 판단해 오케스트레이션을 수행합니다. 본 교육의 핵심이 바로 이 "AI에게 오케스트레이션을 맡기는 법"입니다.

Anthropic은 Claude를 세 가지 체급으로 나누어 제공합니다. 이름은 시(詩)에서 따왔으며, 순서대로 짧은 시(빠름) → 긴 시(균형) → 서사시(깊이)의 은유입니다.

Appendix · Troubleshooting

06자주 발생하는 오류 FAQ

설치 중 자주 만나는 문제들과 해결 방법입니다. 문제가 생기면 먼저 여기를 확인해 주세요.

Q1. claude: command not found 또는 "인식할 수 없는 명령어"

원인. 설치는 됐지만 터미널이 아직 새 실행 파일 경로(PATH)를 모르고 있습니다. 특히 Windows PowerShell에서 자주 발생합니다.

1단계 — 가장 간단한 방법

터미널(또는 PowerShell) 창을 모두 닫고 완전히 새로 열어 주세요. 대부분 이것으로 해결됩니다. 이래도 안 되면 컴퓨터를 재부팅하신 뒤 다시 시도합니다.

2단계 — Windows: 설치 자체가 되었는지 먼저 확인 Windows

PowerShell을 열고 아래 명령으로 실행 파일이 해당 위치에 존재하는지부터 확인합니다.

PS> Test-Path "$env:USERPROFILE\.local\bin\claude.exe"
  • 결과가 True → 설치는 됐지만 PATH 문제입니다. 3단계로 진행하세요.
  • 결과가 False → 설치 자체가 실패했습니다. Part 2.2의 설치 명령을 처음부터 다시 실행하십시오.

3단계 — Windows: PATH 수동 등록 (가장 확실) Windows

아래 두 가지 방법 중 편한 쪽을 선택하시면 됩니다.

방법 A. PowerShell 한 줄로 영구 등록 (권장)

아래 명령 한 줄을 PowerShell에 붙여 넣으면 사용자 환경변수 Path에 Claude Code 설치 경로가 영구적으로 추가됩니다.

PS> [Environment]::SetEnvironmentVariable("Path", [Environment]::GetEnvironmentVariable("Path","User") + ";$env:USERPROFILE\.local\bin", "User")

실행 후 PowerShell 창을 완전히 닫고 새로 열어 주세요. 그런 다음 검증:

PS> claude --version

버전 번호가 표시되면 해결입니다.

방법 B. Windows 설정 화면에서 수동으로

명령어 입력이 부담스러우시면 GUI로 설정하실 수 있습니다.

  1. 시작 메뉴에서 "환경 변수"라고 검색 → "사용자 계정의 환경 변수 편집" 선택
  2. 위쪽 "사용자에 대한 사용자 변수" 목록에서 "Path"를 선택하고 "편집" 클릭
  3. 우측 "새로 만들기" 클릭 후 아래 경로를 정확히 입력 → "확인"
    %USERPROFILE%\.local\bin
  4. 모든 창에서 "확인"을 누른 뒤 PowerShell을 완전히 닫고 새로 열기
  5. claude --version 으로 확인

3단계 — macOS: 쉘 프로파일에 경로 추가 macOS

macOS에서 재시작해도 인식이 안 된다면, 기본 쉘(zsh) 프로파일에 Claude 설치 경로가 빠져 있을 수 있습니다. 아래 한 줄을 터미널에 붙여 넣으면 됩니다.

$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc $ source ~/.zshrc $ claude --version

버전이 표시되면 해결입니다. (bash를 쓰신다면 ~/.zshrc 대신 ~/.bash_profile에 추가하세요.)

⚠️ 그래도 안 될 때. PowerShell의 실행 정책이 막혀 있을 수 있습니다. 관리자 권한 PowerShell을 새로 열고 Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser를 한 번 실행한 뒤 Y로 허용하신 다음 다시 시도해 주세요.

Q2. Windows에서 'irm' is not recognized as an internal or external command

원인. 현재 창이 PowerShell이 아닌 CMD(명령 프롬프트)입니다.

해결. 시작 메뉴에서 "PowerShell"을 검색해 새로 실행하세요. 프롬프트 맨 앞에 PS가 붙어 있어야 정상입니다.

Q3. Windows에서 Python 실행 시 Microsoft Store가 열림

원인. Python 설치 시 "Add python.exe to PATH" 체크박스를 놓쳤습니다.

해결. Python 설치 프로그램을 다시 실행해 "Modify" → 다음 화면에서 "Add Python to environment variables"에 체크한 뒤 재설치합니다. 그래도 안 되면 설치를 "Uninstall"하고 처음부터 다시 설치하시는 것이 가장 확실합니다.

Q4. claude mcp list를 해도 playwright/context7이 보이지 않음

원인. --scope user 옵션 없이 설치되어 해당 폴더에서만 등록됐거나, 설치 명령이 실패했습니다.

해결. 같은 폴더에서 claude mcp list로 확인하고, 없으면 Part 3의 설치 명령을 다시 실행하세요. 반드시 --scope user가 포함되어야 모든 폴더에서 쓸 수 있습니다.

Q5. Playwright MCP 사용 시 "브라우저를 설치해야 한다"는 메시지

해결. Claude Code 밖 터미널에서 아래 명령으로 브라우저를 수동 설치합니다. (약 500MB 다운로드)

$ npx playwright install chromium

Q6. macOS에서 "개발자를 확인할 수 없어 열 수 없습니다"

해결. 시스템 설정 → 개인정보 보호 및 보안 → 화면 하단 "확인 없이 열기"를 클릭하면 해당 앱이 실행 허용됩니다.

Q7. 설치는 됐는데 Claude에 로그인하니 "구독이 필요합니다"라는 메시지

원인. 무료 계정으로 Claude Code를 사용하려 했습니다.

해결. claude.ai에 접속해 Pro 이상 요금제를 구독한 뒤, 터미널에서 claude를 다시 실행해 재로그인하시면 됩니다.

Q8. 회사 네트워크에서 설치가 자꾸 실패합니다

원인. 회사 방화벽·프록시가 외부 다운로드를 차단하고 있을 수 있습니다.

해결. IT 담당자에게 다음 도메인으로의 접근 허용을 요청하세요: claude.ai, api.anthropic.com, nodejs.org, registry.npmjs.org, github.com, python.org, mcp.context7.com. 개인 핫스팟(휴대폰 테더링)으로 일단 설치하는 것도 방법입니다.

그래도 해결되지 않는다면. 교육 담당자에게 오류 메시지 전문을 스크린샷으로 찍어 보내 주세요. 교육 시작 최소 24시간 전까지 문의해 주시면 당일에 다 함께 해결할 수 있습니다.
